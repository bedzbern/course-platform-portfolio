# Lesson 00: Welcome & Getting Started

Welcome to **Manga Portfolio Builder** — a complete course where you'll build a real portfolio website from scratch using nothing but HTML, CSS, and JavaScript.

By the end of this course, you'll have a live website on the internet that looks like a manga/comic-book themed portfolio.

---

## What You Need

| Tool | Why You Need It |
|------|----------------|
| **VS Code** (or any text editor) | To write your code |
| **A web browser** (Chrome, Firefox, Edge) | To view your site |
| **A GitHub account** (free) | To put your site live (Phase 6) |
| **Git** (optional for Phase 6) | To push code to GitHub |

If you don't have VS Code, download it for free at https://code.visualstudio.com/

## How the Course Works

### 6 Phases, 37 Lessons

| Phase | What You Build | Lessons |
|-------|----------------|---------|
| 1 — HTML | Page structure (ugly but complete) | 10 |
| 2 — CSS | Colors, fonts, grid layout | 10 |
| 3 — Manga Effects | Badges, speech bubbles, ink stamps | 5 |
| 4 — JavaScript | Scroll effects, form handling | 4 |
| 5 — Evolution | Dynamic pages, JS automation | 5 |
| 6 — Ship | SEO, responsive polish, deploy | 3 |

### Each Lesson Has

1. **Goal** — one clear thing you'll learn
2. **What You Type** — code to type yourself (no copy-paste!)
3. **What Each Line Means** — explanation table
4. **Try It** — a small challenge to reinforce learning
5. **What You Learned** — checklist of concepts

### Snapshots

After each phase, there's a **snapshot** folder (`snapshots/phase-N-name/`) that contains a complete working copy of what your site should look like at that point. Open the `index.html` in your browser to compare.

---

## The Golden Rule: Type Every Line

The most important rule of this course:

> **Type every line of code yourself. Do not copy-paste.**

Typing builds:
- **Muscle memory** — your fingers learn the syntax
- **Attention to detail** — you catch typos and understand structure
- **Confidence** — you know you wrote it and understand it

When you get stuck, check the snapshot. Then fix it yourself.

## Your Project Folder

All your work goes in the `portfolio/` folder. The root `portfolio/` folder contains the final answer key — but you'll be building your version alongside it.

```
portfolio/
├── index.html         ← Your main page
├── style.css          ← Your styles
├── script.js          ← Your JavaScript
├── project.html       ← Dynamic detail page
├── course/            ← These lessons
└── snapshots/         ← Reference copies at each phase
```
